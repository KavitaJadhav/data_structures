HASH TABLE / HASH MAP – NOTES

Definition:
A hash table is a data structure that stores key-value pairs.
It uses a hash function to compute an index into an array of buckets.

Properties:
- Keys are unique.
- Provides fast access (search, insert, delete) in average case.
- Can handle collisions using chaining or open addressing.

HASH FUNCTION:
- Maps key → integer index.
- Should distribute keys uniformly.
- Example: hash(key) = key % table_size

COLLISION HANDLING:
1. Chaining
   - Each bucket stores a linked list of entries.
   - Complexity:
     Average: O(1)
     Worst: O(n) (all keys in one bucket)
2. Open Addressing
   - Find next empty slot (linear probing, quadratic probing, double hashing)
   - Complexity:
     Average: O(1)
     Worst: O(n) (if table nearly full)

OPERATIONS:

Operation        Description                 Complexity (average)
---------        -----------                 ------------------
Insert / Put     Add key-value pair          O(1)
Search / Get     Retrieve value by key       O(1)
Delete / Remove  Remove key-value pair       O(1)

Worst-case: O(n) (all keys collide)

EXAMPLES:

Java HashMap:
Map<Integer, String> map = new HashMap<>();
map.put(1, "A");
map.put(2, "B");
map.get(1);        // returns "A"
map.remove(2);

Python dict:
d = {}
d[1] = "A"
d[2] = "B"
print(d[1])        # "A"
del d[2]

INTERVIEW TIP:
- Average case: O(1) for insert, search, delete
- Worst case: O(n) due to collisions
- Choose a good hash function to minimize collisions
- Be familiar with chaining vs open addressing

HASH TABLE COLLISION RESOLUTION – NOTES

When two keys hash to the same index → collision occurs.
Collision resolution techniques:

1️⃣ Linear Probing
- Idea: If bucket[i] is occupied, check the next bucket (i+1), then (i+2), and so on until an empty slot is found.
- Formula:
  index = (hash(key) + i) % table_size, i = 0, 1, 2, ...
- Pros: Simple, easy to implement
- Cons: Clustering problem (primary clustering)
- Time Complexity (average): O(1)
- Time Complexity (worst): O(n)

2️⃣ Quadratic Probing
- Idea: Instead of checking consecutive buckets, jump quadratically: i^2.
- Formula:
  index = (hash(key) + i^2) % table_size, i = 0,1,2,...
- Pros: Reduces primary clustering
- Cons: Can still have secondary clustering
- Time Complexity: O(1) average, O(n) worst
- Requirement: Table size should be prime to ensure all slots probed

3️⃣ Double Hashing
- Idea: Use a **second hash function** to compute step size for probing.
- Formula:
  index = (hash1(key) + i * hash2(key)) % table_size
- Pros: Avoids clustering
- Cons: Slightly more computation
- Time Complexity: O(1) average, O(n) worst
- Requirement: hash2(key) ≠ 0, typically table size is prime

---

EXAMPLES:

Linear Probing:
Hash table size = 7
Insert keys: 10, 17
hash(key) = key % 7
10 → index 3
17 → index 3 occupied → try 4 → insert at 4

Quadratic Probing:
index = (hash(key) + i^2) % 7
10 → 3
17 → 3 occupied, i=1 → 3+1=4 → empty → insert

Double Hashing:
hash1(key) = key % 7
hash2(key) = 1 + (key % 5)
10 → index = 10%7 = 3
17 → index = 17%7=3, step = 1 + 17%5 = 1+2=3
next index = (3 + 1*3)%7 = 6 → insert at 6
